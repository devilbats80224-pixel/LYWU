角色說明

建單端（intake）：建立申請單、送出 create_order 事件、接收收單端回寫狀態；可執行本人刷卡確認結束。一般不負責開始/電話結束。
收單端（receiver）：自動收單、開始作業、記錄電話通知結束、本人確認結束、編輯追加，並回寫狀態事件。
混合模式（hybrid）：同時具備兩邊能力，僅建議測試或備援使用，正式上線不建議長期使用。

系統說明

1. 本系統完全離線運作，不依賴任何外部資源，也不會主動連網。
2. 資料僅儲存在本機瀏覽器（IndexedDB + localStorage 雙重自動保存），請定期匯出完整備份 JSON。
3. 本系統支援終端角色模式：建單端 intake / 收單端 receiver / 混合模式 hybrid（同一份 HTML，依本機設定切換）。
4. 雙機同步使用「共享資料夾事件檔」：每次操作輸出獨立事件 JSON，另一端掃描後合併，不走外網 API。
5. 第一次使用同步功能需在管理區手動授權同步資料夾，並建立 events / archive / errors 子資料夾。
6. 自動掃描依賴瀏覽器本機資料夾授權能力（建議 Chromium 系）；若瀏覽器不支援，離線核心功能仍可正常使用。
7. 完整備份 JSON 僅用於備份/還原/換機，不作為雙機即時同步機制。
8. 員工資料可在管理區手動新增/編輯，或匯入 CSV / XLSX；刷卡/工號輸入後可自動帶出姓名、部門、電話。
9. 本人確認結束流程：申請人電話通知結束後，需回辦公室再刷卡確認，系統才會轉為「已完成」。
10. 隔離點位是條件式欄位：只有選擇「需要隔離點位」時才必填，不是每張單都必填。
11. 管理模式用途：可代人刷卡確認結束、刪除員工資料、刪除工單資料與執行高風險維護操作。
12. 若非原申請人確認結束，需先開啟管理模式，且必填代確認原因；系統會完整記錄代確認資訊。
13. 工單在進行中可執行「編輯追加」，補登作業項目與隔離點位，避免重複建單。
14. 同一張工單可執行「補簽到」：設備工程師與廠務工程師可分次補登，並可同時補正機台/盤面資訊。
