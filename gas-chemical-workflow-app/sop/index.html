送氣／送化作業申請與隔離管理系統

SOP 操作手冊（圖文版）

適用版本：gas_chemical_workflow_offline.html（2026-04-16 版）
文件日期：2026-04-16
文件目的
本手冊用於辦公室值班人員、設備工程師、廠務工程師之日常操作，涵蓋建單、流程控管、補簽到、完整編輯、同步設定、資料備份還原。
重要原則
1. 本系統為離線單機 HTML，資料儲存在本機瀏覽器。
2. 若有雙機協作，請使用「終端角色 + 共享資料夾事件同步」，不要用完整備份 JSON 當同步機制。
3. 高風險操作（代確認、刪除）請先開啟管理模式。

1. 快速上手流程（建議）

  1. 先看角色狀態：確認目前是建單端/收單端/混合模式。
  2. 新增申請：填申請人、作業項目、隔離需求，送出建單。
  3. 歷史作業列表追蹤：依狀態操作開始、通知結束、本人確認或編輯。
  4. 補簽到 / 完整編輯：同一張單可補第二位工程師，並修正錯填資料。
  5. 管理區維護：員工主檔、同步設定、匯入/匯出、備份還原。

2. 首頁 Dashboard 操作

2.1 狀態總覽卡片

  • 卡片顯示：待開始、進行中、待本人確認、已完成、已取消。
  • 可直接點卡片，快速篩選下方記錄。
  • 再點一次同卡片可取消篩選。

2.2 快速搜尋

  • 可搜尋機台/盤面/姓名/工號/電話/品項/點位/備註。
  • 適合快速定位特定單號與責任人。

3. 新增申請（簽到建單）

3.1 必填邏輯

  • 申請人（工號/姓名/部門/電話）
  • 作業類型
  • 至少一筆作業項目（品項、數量>0）
  • 機台或盤面至少一個

3.2 第二簽到人（可選）

  • 可在建單當下直接填入第二位（設備/廠務）一起簽到。
  • 若查無員工主檔，可手填姓名/部門/電話，系統可自動建檔。

3.3 隔離點位與項目對應

  • 每一筆作業項目可填「對應偵測器點位（可選）」。
  • 若該 gas/chemical 不需點位，可留空。
  • 「需要隔離點位」= 是 時，仍需至少一筆點位（可由項目映射自動帶入）。

4. 歷史作業列表（原今日作業列表）

4.1 查詢與篩選

  • 日期可留空查全部，或指定日期。
  • 狀態、作業類型、是否隔離可交叉篩選。
  • 表格為資訊濃縮排版：機台/盤面、人員資訊、作業資訊、時間紀錄、操作按鈕。

4.2 流程按鈕（依角色限制）

狀態常用操作說明
待開始開始 / 完整編輯 / 查看詳情由收單端主責開始流程
進行中已通知結束 / 編輯追加 / 完整編輯通知結束後轉待本人確認
待本人確認本人確認結束 / 補簽到 / 完整編輯本人刷卡確認或管理模式代確認
已完成/已取消完整編輯 / 查看詳情可修正資料但不再走主流程

5. 工單完整編輯（含設備/廠務資料修正）

紅框編號對照：請依圖中 1 → 2 → 3 → 4 順序操作。

5.1 可修正欄位

  • 紅框 1：申請人基本資料與作業型態。
  • 紅框 2：設備/廠務簽到人（工號、姓名、部門、電話）。
  • 紅框 3：隔離設定、作業項目與對應偵測器點位。
  • 紅框 4：按「儲存完整編輯」完成覆蓋更新。
  • 申請人：工號、姓名、部門、電話、角色
  • 設備工程師：工號、姓名、部門、電話
  • 廠務工程師：工號、姓名、部門、電話
  • 機台/盤面、預計時程、備註
  • 作業項目（含每筆對應點位）與隔離點位清單

5.2 編輯注意事項

  • 這是覆蓋式儲存，按下「儲存完整編輯」會以本次內容更新工單。
  • 若設備/廠務改人或改聯絡資訊，簽到時間會依變更規則更新。
  • 請確認部門/電話正確，避免後續通知找不到人。

6. 補簽到（第二位工程師）

紅框編號對照：補簽到也走完整編輯視窗，請依 1 → 2 → 3 → 4 操作。

6.1 使用時機

  • 同一台機台已有工單，但後續設備/廠務另一方補簽到。
  • 避免同機台重複開第二張單造成追蹤混亂。

6.2 操作重點

  • 紅框 1：先填第二位工程師申請資料（可刷卡/工號查詢帶入）。
  • 紅框 2：補齊設備/廠務簽到資訊。
  • 紅框 3：必要時同步補正時程、隔離點位與作業項目。
  • 紅框 4：按儲存完成補簽到。
  • 補簽到可選設備或廠務角色。
  • 支援刷卡/工號查詢，查無員工時可手填並建檔。
  • 可同步補正機台/盤面、補登點位，必要時可切到完整編輯調整項目。

7. 管理區總覽

7.1 員工主檔管理

  • 手動新增/編輯（含在職切換）。
  • 支援 CSV / XLSX 匯入。
  • 建單/補簽到查無員工時，可即時補建檔。

7.2 備份與還原

  • 完整備份 JSON：災難復原、換機遷移。
  • 完整查詢結果 XLSX：稽核、報表、跨日期資料整理。

8. 終端角色與同步設定（雙機協作）

8.1 角色定義

  • 建單端（intake）：建立申請、補簽到、完整編輯、本人確認。
  • 收單端（receiver）：主流程操作（開始、通知結束、確認、取消等）。
  • 混合模式（hybrid）：測試/備援，正式上線不建議長期使用。

8.2 同步設定步驟

  1. 管理區選擇終端模式與終端名稱。
  2. 按「選擇同步根資料夾」，完成授權。
  3. 確認子資料夾狀態：events / archive / errors。
  4. 啟用自動掃描並設定掃描秒數（建議 5~10 秒）。
  5. 觀察「最近同步紀錄」與匯入/匯出統計。

9. 通知與簽退管理建議

建議值班追蹤方式
在歷史作業列表同時查看「設備工程師 / 廠務工程師」欄位與「時間紀錄」，以避免漏提醒簽退。
  • 若作業到「待本人確認」，請主動通知人員回來刷卡。
  • 若發現資料有誤，優先用「完整編輯」修正，避免重複開單。
  • 若兩邊人員分開到場，優先用「補簽到」補第二位工程師。
  • 若要查跨日資料，使用歷史作業列表日期篩選 + XLSX 匯出。

10. 常見問題（FAQ）

Q1：查無員工怎麼辦？

A：可直接手填姓名/部門/電話，送單或補簽到時會自動建檔；也可到管理區手動建立。

Q2：隔離點位一定要填嗎？

A：不是。只有在「需要隔離點位 = 是」時才強制至少一筆。每項目的對應點位可個別留空。

Q3：設備/廠務資料填錯可改嗎？

A：可以，進入「完整編輯」可修正設備/廠務的工號、姓名、部門、電話。

Q4：雙機協作為何不用完整備份 JSON 同步？

A：完整備份是還原用途；雙機協作應用共享資料夾事件檔（append-only）避免互覆蓋。

Q5：資料安全如何做？

A：至少每日一次匯出完整備份 JSON，並定期匯出完整查詢結果 XLSX 供稽核追溯。
