施工管制表整理工具
使用教學 SOP（依實際工作流）

版本日期：2026-04-17
目的：每日上傳報表後，快速盤點、校正、匯出並貼回公版
本版重點：移除「合併新報表 / 套用整理功能」主流程，改成待檢查與批次確認 Site 為核心

目錄

  1. 前置說明與本版範圍
  2. 標準操作流程（SOP）
  3. 篩選邏輯與底色規則
  4. 每日檢核清單

1. 前置說明與本版範圍

  • 本 SOP 依你目前日常流程撰寫：上傳 → 待檢查盤點 → 批次確認Site → 全表複檢 → 下載/複製 → 貼回公版。
  • 「合併新報表（增量更新）」與「套用整理功能按鈕」不列入主流程（可保留在工具中，但本 SOP 不當成每日必做）。
  • 下方所有圖都用紅框與中文步驟字標示重點操作區。
建議先用同一份來源檔試跑一遍，確認欄位、底色與公版貼值一致，再投入正式班別資料。

2. 標準操作流程（SOP）

Step 1

上傳來源報表並確認解析成功

  1. 在上傳區放入 ESP 匯出的施工報表。
  2. 確認畫面已載入資料列與欄位數。
  3. 先看是否有明顯異常（例如筆數明顯不對）。
Step 2

用篩選按鈕集中盤點「待檢查」異常並人工修正

  1. 先按「待檢查」，把需要人工確認的工項集中顯示。
  2. 再交叉搭配 Site（P1~P4）、高風險、夜間、簽到簽退、巡檢按鈕做分群盤點。
  3. 在表格內直接修改欄位值，修完再切下一組篩選。

篩選按鈕可同時開多個，邏輯為交集（同一列需同時滿足所有啟用條件）。

按鈕 / 群組預設篩選條件（依工具程式）
篩選設定可編輯 ISEP / PUSD / CCD / EBO 等群組關鍵字清單，改完後按鈕即套用新規則。
ISEP課別符合 ISEP 群組值（預設：F20ISEP1；可在「篩選設定」調整）。
PUSD課別符合 PUSD 群組值（預設：PUSD 廠區服務一課；可調整）。
CCD課別符合 CCD 群組清單（資料中心相關課別，清單可調整）。
EBO課別符合 EBO 群組清單（EBO Clean / Process / Repair…，清單可調整）。
P1 / P2 / P3 / P4Phase(Site) 文字包含對應碼；例如 Outsite,P1 也會命中 P1。
廠務L1高風險作業 欄位包含 L1。
廠務L2高風險作業 欄位包含 L2。
設備高風險高風險作業 欄位包含 設備高風險。
夜間施工簽退時間命中 18:30~23:59 或 00:00~00:30；簽退空值也納入。
待檢查優先抓黃底旗標（ISEP/Site/申請類別）；若旗標欄不存在，退回即時計算規則。
消防中斷/火警隔離申請類別命中消防中斷/火警隔離（含代碼 8/10）或欄位值為 V。
尚未分類SitePhase 非單獨 P1/P2/P3/P4（空值、多值、Outsite 皆視為未分類）。
已簽到 / 未簽到簽到時間 有值 / 空值。
已簽退 / 未簽退簽退時間 有值 / 空值。
已巡檢 / 未巡檢巡檢人員 有值且非 NA / 空值或 NA。
Step 3

修正後進行「批次確認Site」並保存覆寫規則

  1. 完成待檢查項目修正後，按「批次確認Site」集中核定。
  2. 彈窗會列出待核定工項（舊Site→新Site、工程內容、原因/備註），確認後一次送出。
  3. 再視需要匯出 Site 覆寫規則 JSON，留給下一班或隔日沿用。
  4. 此機制目的：遇到同工程名稱 + 施工位置的重複工項時，不必每天重改 Site。
Site 覆寫規則是你的「經驗庫」。同樣條件再次出現時可直接沿用，縮短每日整理時間。
Step 4

全表複檢：先看上方 Site 小表與申請類別，再核對下方統計大表

  1. 上方先看 Site 小表，再展開申請類別下拉做交叉檢查。
  2. 下方大表：施工件數統計 + 日/夜巡檢 + 單位統計。
  3. 若你手動改了申請類別或其他關鍵欄位，上下統計大小表都會同步聯動變化。
Step 5

輸出整理結果，或一鍵複製 ISEP~消防中斷欄位

  1. 先下載 下載處理結果.xlsx 當完整備份。
  2. 若只需目前篩選群組，可用 下載篩選處理結果.xlsx。
  3. 需要貼回公版時，可用 複製 ISEP~消防中斷欄位，並搭配下方統計大表件數一起貼回。
Step 6

貼回公版報表：先清空資料列與底色，再貼回整理結果（公版檔示範）

  1. 開啟公版施工報表。
  2. 保留表頭與版型，先清空舊資料列內容與底色。
  3. 新增足夠空白資料列。
  4. 貼上網頁整理好的所有工項（包含底色）。
  5. 再貼上網頁統計大表對應件數到公版統計區。
若已手動改申請類別，請務必用最新統計結果覆蓋公版統計區，不要沿用舊件數。

3. 篩選與底色規則（公版一致）

3.1 底色規則

  • 廠務 L1 + 設備高風險：紅色底
  • 廠務 L2：黃色底
  • NA + 高風險場所：灰色底

你網頁整理好的資料列底色邏輯，需與公版報表保持一致，貼回時連同底色一起貼。

3.2 本 SOP 未納入主流程的功能

  • 合併新報表（增量更新）
  • 套用整理功能按鈕（排序/GMS/規則按鈕的逐一操作）

以上兩項仍可作為特殊情境備用，但不是你目前每天必做步驟。

4. 每日檢核清單

  • 來源報表上傳後，先確認筆數/欄位數合理。
  • 待檢查 + Site/風險/簽到簽退/巡檢交叉盤點是否完成。
  • 批次確認Site是否已核定，必要時是否已匯出覆寫規則 JSON。
  • 上方統計小表與下方統計大表是否一致聯動。
  • 下載結果是否完成，且公版已先清空資料列/底色再貼回。
  • 公版統計區件數是否已使用最新聯動結果覆蓋。

本文件可直接列印成 PDF；每次工具規則調整後，建議同步更新這份 SOP。
